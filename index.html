Parsly: dev@parsly.com URL: http://parsly.com

Parsly services for invoice documents and analysis data. Uploaded invoice documents are processed by a invoice parser engine, and the resulting analysis data is then made available for each document.

The dropzone service provide endpoints for uploading invoice files. Supported upload formats are:

  • application/pdf
  • image/jpg
  • image/png

A file can either describe one invoice, or part of a invoice document. Dropzone expects all files needed to describe a complete invoice when called, whereby it will create a document resource to keep track of all related source files. Exactly how the files are uploaded depend on which endpoint is used.

Invoice documents are processed asynchronouysly, and to determine when the analysis result is available, you should poll their status via the Pickup service. Document processing status are:

Status Description
Pending Resource created but file sources not yet uploaded.
Uploaded All file sources have been uploaded.
Compiled Available file sources compiled into a single document object.
Processed Invoice document processed by the parser engine.
Invalid Document is not a valid invoice.

When a file reaches the Processing status, the corresponding analysis data will be available via the Pickup service.

Pickup

Facilities to manage documents and retrieve invoice parser results.

List Uploaded Documents.

List uploaded documents. Note that the response is paginated, so the list may not contain all available documents. Use the pageSize query parameter to change the number of documents to include in the list, and use the pageId query parameter to step through the available pages. When the list is empty, the last page has been reached. The total number of available documents is available via the X-Total-Count response header.
