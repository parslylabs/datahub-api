Parsly API

Parsly: dev@parsly.com URL: http://parsly.com

Parsly services for invoice files and analysis data. Uploaded invoice files are processed by a invoice parser engine, and the resulting analysis data is then made available for each file.

When you upload a file, a file resource is created to keep track of its payload, processing status, and resulting analysis data. As the invoice parser engine processes invoices asynchronously, you should poll the processing status of the corresponding files to determine when the result is available.

Related invoice files are uploaded as a logical group to the Dropzone service. A file group resource is created to identify the related files, which you can then use to lookup the processing status of all files via a single request to the Pickup service. The file processing status are:

Status Description
Pending Resource created but payload not yet uploaded.
Uploaded Invoice uploaded.
Processed Invoice processed by the parser engine.
Invalid File is not a valid invoice.

When a file reaches the Processing status, the corresponding analysis data will be available via the Pickup service.

required

One or more invoice files via Multipart-Formdata content type.

Array of strings <binary> [ items <binary > ]

Form field containing an invoice encoded as binary data and limited to the following content types:

  • image/jpg

Pickup

Facilities to manage file groups and retrieve invoice parser results.

List Uploaded File Groups.

List uploaded file groups. Note that the response is paginated, so the list may not contain all available file groups. Use the pageSize query parameter to change the numer of file groups to include in the list, and use the pageId query parameter to step through the available pages. When the list is empty, the last page has been reached. The total number of available file groups is available via the X-Total-Count response header.

Authorizations:
ApiKeyHeader
query Parameters
pageId
integer >= 1
Default: 1

ID of page to request (count starts from 1).

pageSize
integer [ 10 .. 50 ]
Default: 20

Number of items to include per page.
